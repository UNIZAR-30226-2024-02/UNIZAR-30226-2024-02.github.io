API Sockets

Las API Sockets se utilizan para crear aplicaciones en tiempo real. Se utilizan para crear una conexión entre el cliente y el servidor. El servidor puede enviar datos al cliente y el cliente puede enviar datos al servidor. Esto es útil para aplicaciones que requieren actualizaciones en tiempo real, como aplicaciones de chat, juegos en línea y transmisión en vivo.

Las API Sockets utilizan el protocolo WebSocket, que es un protocolo de comunicación que proporciona canales de comunicación full-duplex sobre una única conexión TCP. Esto permite una comunicación de baja latencia entre el cliente y el servidor.

Las API Sockets se utilizan comúnmente en conjunto con API Web y API Controllers para crear una aplicación web completa. El cliente puede usar API Web para hacer solicitudes HTTP al servidor, y el servidor puede usar API Controllers para procesar esas solicitudes y enviar datos de vuelta al cliente usando API Sockets.

Las API Sockets se implementan utilizando la biblioteca Socket.IO, que es una biblioteca de JavaScript que proporciona una API sencilla para crear aplicaciones web en tiempo real. Socket.IO maneja los detalles de bajo nivel del protocolo WebSocket y proporciona una API de alto nivel para enviar y recibir mensajes entre el cliente y el servidor.

Las API Sockets son una herramienta poderosa para crear aplicaciones web en tiempo real. Permiten una comunicación rápida y eficiente entre el cliente y el servidor, y se pueden usar para crear una amplia gama de aplicaciones, desde aplicaciones de chat hasta juegos en línea hasta servicios de transmisión en vivo.

Sesión de usuario

Emit

Evento 'login'

El evento 'login' se emite desde el cliente al servidor utilizando la función 'emit' de Socket.IO.

Uso
socket.emit('login', userId)
Parámetros
  • userId: El ID del usuario que intenta iniciar sesión.
Descripción

Este código en JavaScript utiliza la función 'emit' de Socket.IO para enviar un evento llamado 'login' desde el cliente al servidor. El evento 'login' lleva un objeto de datos que tiene una propiedad 'userId' con el valor del ID del usuario que está intentando iniciar sesión.

El evento 'sendChatMessage' se emite desde el cliente al servidor utilizando la función 'emit' de Socket.IO.

Parámetros

  • user: El usuario que envía el mensaje.
  • timestamp: La marca de tiempo en la que se envía el mensaje.
Descripción

Este código en JavaScript utiliza la función 'emit' de Socket.IO para enviar un evento llamado 'sendChatMessage' desde el cliente al servidor. El evento 'sendChatMessage' lleva un objeto de datos que incluye las propiedades 'chatId', 'message', 'user' y 'timestamp', que representan respectivamente el ID del chat, el contenido del mensaje, el usuario que envía el mensaje y la marca de tiempo del mensaje.

On

Evento 'chatMessage'

El evento 'chatMessage' se emite desde el servidor al cliente utilizando la función 'emit' de Socket.IO.

Uso
socket.on('chatMessage', (body) => { // Lógica para manejar el mensaje del chat });
Parámetros
  • body: Un objeto que contiene los detalles del mensaje del chat.
Descripción

Este código en JavaScript utiliza la función 'on' de Socket.IO para escuchar un evento llamado 'chatMessage' enviado desde el servidor al cliente. El evento 'chatMessage' lleva un objeto de datos llamado 'body' que incluye las siguientes propiedades:

  • chatId: El ID del chat en el que se envía el mensaje.
  • message: El contenido del mensaje que se ha enviado.
  • user: El usuario que ha enviado el mensaje.
  • timestamp: La marca de tiempo en la que se ha enviado el mensaje.

En el código de ejemplo, el evento 'chatMessage' se maneja mediante una función que recibe el objeto 'body'. Dentro de esta función, se puede definir la lógica necesaria para procesar y mostrar el mensaje del chat en el cliente.

Evento 'ataco'

El evento 'ataco' se emite desde el cliente al servidor utilizando la función 'emit' de Socket.IO.

Uso
socket.emit('ataco', { userOrigen, userDestino, dadosAtacante, dadosDefensor, tropasPerdidasAtacante, tropasPerdidasDefensor, conquistado, territorioOrigen, territorioDestino, eloAtacante, eloDefensor, dineroAtacante, dineroDefensor, clientIp });
Parámetros
  • userOrigen: El usuario que está atacando.
  • userDestino: El usuario que está siendo atacado.
  • dadosAtacante: Los dados utilizados por el atacante.
  • dadosDefensor: Los dados utilizados por el defensor.
  • tropasPerdidasAtacante: El número de tropas perdidas por el atacante.
  • tropasPerdidasDefensor: El número de tropas perdidas por el defensor.
  • conquistado: Indica si el territorio ha sido conquistado.
  • territorioOrigen: El territorio desde el cual se origina el ataque.
  • territorioDestino: El territorio que está siendo atacado.
  • eloAtacante: La puntuación ELO del atacante.
  • eloDefensor: La puntuación ELO del defensor.
  • dineroAtacante: La cantidad de dinero del atacante.
  • dineroDefensor: La cantidad de dinero del defensor.
  • clientIp: La dirección IP del cliente.
Descripción

Este código en JavaScript utiliza la función 'emit' de Socket.IO para enviar un evento llamado 'ataco' desde el cliente al servidor. El evento 'ataco' lleva un objeto de datos con varias propiedades que describen los detalles del ataque en un juego, incluyendo los usuarios involucrados, los resultados de los dados, las tropas perdidas, y los territorios afectados.
